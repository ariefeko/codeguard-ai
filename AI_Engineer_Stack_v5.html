✓ CoreLocal Inference

# Test cepat
ollama run qwen2.5-coder:7b "Tulis hello world Python"

Git Safety

✓ CriticalIsolation

Git mencatat semua perubahan — tapi branch memberikan isolasi. Bukan soal recording, tapi soal "dapur terpisah" agar eksperimen AI tidak merusak main branch.

git checkout main && git branch -D ai/nama-task

🏖️ 5.2 Sandbox Philosophy

OpenCode menjalankan semua aksi AI di dalam Docker container terpisah — bukan di sistem kamu langsung. Ini adalah desain keamanan fundamental.

Kenapa Sandbox?
  • Isolation: AI tidak bisa sentuh file di luar project folder
  • Reproducible: environment sama setiap run
  • Rollback mudah: destroy container = semua bersih
  • No pollution: dependencies AI tidak masuk ke sistem kamu
Host System (kamu)↔ hanya expose project folder
│
OpenCode Containeragent logic, UI, orchestration
│
Sandbox Containereksekusi kode, terminal, browser
│
Ollama (host)localhost:11434 ← model inference
🔌 5.3 Model Connection

Konfigurasi OpenCode — buka terminal di direktori project-mu, lalu:

config — OpenCode Settings

⚠️ host.docker.internal

Dari dalam Docker container, localhost merujuk ke container itu sendiri — bukan host kamu. Gunakan host.docker.internal untuk reach Ollama yang jalan di host. Di Linux kadang perlu tambahkan --add-host=host.docker.internal:host-gateway ke docker run.

Settings → Agent → Confirmation Mode: ON. Dengan ini, setiap kali AI mau jalankan command atau hapus file, kamu harus approve dulu. Sangat direkomendasikan saat belajar.

💥 5.6 Failure Handling

AI akan gagal. Ini normal. Yang penting adalah bagaimana kamu merespons:

Tanda AI Sedang Halu
  • Import library yang tidak ada
  • Loop tanpa progress (stuck)
  • Error yang sama terus berulang
  • Output tidak sesuai task sama sekali
Cara Handle
  • Stop agent, jangan biarkan jalan terus
  • Review perubahan dengan git diff
  • Rephrase task — lebih spesifik
  • Pecah task besar jadi sub-task kecil
bash — cek apa yang sudah diubah AI
# Lihat semua perubahan sejak baseline
git diff HEAD

# Lihat file apa saja yang berubah
git status

# Lihat history commit AI
git log --oneline -10
⏪ 5.7 Git Rollback Strategy
bash — Rollback Playbook
── Skenario 1: AI baru mulai, belum ada commit ──
git checkout -- .          # batalkan semua perubahan unstaged
git clean -fd              # hapus file baru yang dibuat AI

── Skenario 2: AI sudah commit, mau balik ke baseline ──
git log --oneline          # cari hash commit baseline kamu
git reset --hard abc1234   # balik ke commit itu (hash dari log)

── Skenario 3: Pakai branch (cara terbaik) ──
git checkout main          # kembali ke main yang bersih
git branch -D ai/task      # hapus branch AI sepenuhnya

── Skenario 4: Mau ambil sebagian saja ──
git checkout ai/task -- src/specific_file.py  # cherry-pick file
⚠️ Ingat

git reset --hard menghapus perubahan secara permanen dari working directory. Pastikan kamu sudah git stash atau ada di branch terpisah sebelum melakukan ini.

🔄 5.8 Example Task Lifecycle

Ini adalah workflow lengkap satu task dari awal sampai selesai:

1
Prepare
Buat branch baru: git checkout -b ai/buat-fungsi-parser. Commit baseline: git commit -m "baseline". Aktifkan confirmation mode di OpenCode.
2
Write Task (di chat OpenCode)
Tulis task yang spesifik: "Buat file src/parser.py berisi fungsi parse_csv(filepath) yang menerima path file CSV dan return list of dict. Tambahkan error handling dan type hints."
3
Monitor & Approve
Pantau setiap langkah AI. Approve aksi yang masuk akal, stop jika AI mulai aneh. Perhatikan file apa yang dibuat dan command apa yang dijalankan.
4
Review Output
Baca kode yang dibuat AI. Jalankan sendiri: python src/parser.py. Cek: apakah ada import aneh? apakah ada hardcoded path? apakah logic-nya masuk akal?
5
Decide: Merge atau Rollback
Jika bagus: git checkout main && git merge ai/buat-fungsi-parser. Jika gagal: git branch -D ai/buat-fungsi-parser dan coba lagi dengan task yang lebih spesifik.
💡 Tips dari Pengalaman

Task yang baik untuk AI: spesifik, satu tujuan, ada kriteria sukses yang jelas. Task yang buruk: "buat aplikasi chat" — terlalu luas. Pecah jadi: "buat endpoint POST /message yang terima JSON {text: string} dan return {reply: string}".

Prompt Engineering

Setelah OpenCode★ Core Skill

Setelah merasakan bagaimana AI bekerja di OpenCode, kamu akan mulai menyadari: cara kamu menulis task sangat menentukan hasilnya. Itulah prompt engineering — bukan trik, tapi komunikasi yang presisi.

🏗️ Anatomi Prompt yang Baik
text — Struktur
ROLE        → Kamu adalah senior Python developer...
CONTEXT     → Project ini menggunakan FastAPI + Qdrant...
INSTRUCTION → Buat fungsi yang...
INPUT       → Ini kode existing: [paste]
OUTPUT FMT  → Jawab dengan kode + penjelasan singkat per bagian
CONSTRAINTS → Tanpa external library, type hints wajib
🔧 4 Teknik Utama
Few-shot Prompting

Berikan 2-3 contoh input→output sebelum pertanyaan utama. Konsisten menghasilkan output lebih baik dari zero-shot untuk task spesifik.

Chain of Thought

Tambahkan "Pikirkan langkah demi langkah sebelum menjawab" di akhir prompt. Terbukti meningkatkan akurasi reasoning secara signifikan.

Role Prompting

Spesifik: bukan "kamu programmer" tapi "kamu senior backend engineer, 10 tahun Python, expert distributed systems, selalu tulis kode yang type-safe."

XML Structuring (Khusus Claude)

Claude sangat responsif terhadap tag XML sebagai pemisah context. Gunakan <kode_lama>, <kode_baru>, <konteks> untuk input kompleks.

ParameterNilaiEfek
temperature0.1 untuk kode, 0.7 kreatifDeterminisme vs kreativitas
top_p0.9Keragaman vocabulary
num_ctx4096–32768Panjang context window
repeat_penalty1.1Kurangi pengulangan

Orchestration — LangChain

Butuh Python★ Build Layer

Orchestration adalah layer yang memungkinkan kamu membangun pipeline AI secara programatik — chain prompts, manage memory, parse output terstruktur. LangChain adalah framework paling populer untuk ini.

🦜 Setup LangChain — Hybrid: DeepSeek (cloud) + Ollama (lokal)
bash
pip install langchain langchain-community langchain-ollama langchain-core langchain-openai python-dotenv
bash — .env
# Daftar gratis di platform.deepseek.com
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
python — basic_chain.py
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from dotenv import load_dotenv
import os

load_dotenv()

# LLM → DeepSeek cloud (gratis, cepat, OpenAI-compatible)
model = ChatOpenAI(
    model="deepseek-chat",
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com/v1",
    temperature=0.1
)

prompt = ChatPromptTemplate.from_messages([
    ("system", "Kamu adalah code reviewer. Jawab dalam Bahasa Indonesia."),
    ("human", "Review kode ini:\n{code}\n\nFokus pada: {focus}")
])

chain = prompt | model | StrOutputParser()

result = chain.invoke({
    "code": "def add(a,b): return a+b",
    "focus": "type hints dan docstring"
})
print(result)

RAG System

Butuh Python★ Core Skill

RAG (Retrieval-Augmented Generation) — memberi model AI akses ke dokumen kamu tanpa training ulang. AI menjawab berdasarkan dokumen yang relevan, bukan hanya dari pengetahuan bawaan model.

Dokumen PDF/TXT→ chunking → embedding → Vector DB (Qdrant)
│ (indexing, sekali)
User Query→ embed → similarity search → top-K chunks
│ (retrieval, setiap query)
Query + Chunks → LLM→ grounded answer
bash — dependencies
pip install langchain-community langchain-ollama langchain-openai qdrant-client langchain-qdrant pypdf python-dotenv
ollama pull nomic-embed-text   # embedding tetap lokal — ringan 274MB
docker run -p 6333:6333 qdrant/qdrant
python — rag_system.py
from langchain_community.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_ollama import OllamaEmbeddings
from langchain_openai import ChatOpenAI
from langchain_qdrant import QdrantVectorStore
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough
from dotenv import load_dotenv
import os

load_dotenv()

# 1. Load & Chunk
docs = PyPDFLoader("./data/doc.pdf").load()
chunks = RecursiveCharacterTextSplitter(
    chunk_size=500, chunk_overlap=50
).split_documents(docs)

# 2. Embed → Ollama lokal (ringan, tidak butuh GPU)
embeddings = OllamaEmbeddings(model="nomic-embed-text")
vectorstore = QdrantVectorStore.from_documents(
    chunks, embeddings, url="http://localhost:6333",
    collection_name="my_docs", force_recreate=True
)

# 3. LLM → DeepSeek cloud (gratis, cepat)
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})
llm = ChatOpenAI(
    model="deepseek-chat",
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com/v1",
    temperature=0.1
)

prompt = ChatPromptTemplate.from_template("""
Jawab berdasarkan konteks berikut SAJA.
Jika tidak ada, katakan "Tidak ada informasi."

KONTEKS: {context}
PERTANYAAN: {question}
JAWABAN:""")

rag_chain = (
    {"context": retriever | (lambda docs: "\n\n".join(d.page_content for d in docs)),
     "question": RunnablePassthrough()}
    | prompt | llm | StrOutputParser()
)

print(rag_chain.invoke("Apa isi dokumen ini?"))

API Layer — FastAPI

Butuh Python★ Production Skill

Wrap sistem AI kamu sebagai HTTP service. FastAPI adalah pilihan terbaik karena async-first, auto-documentation, dan type-safe dengan Pydantic.

bash
pip install fastapi uvicorn[standard] pydantic
uvicorn main:app --reload --port 8000
# Docs otomatis tersedia di: http://localhost:8000/docs
python — main.py
from fastapi import FastAPI, HTTPException
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage
from dotenv import load_dotenv
import time, os

load_dotenv()
app = FastAPI(title="AI Service — DeepSeek + Ollama")

# LLM hybrid: DeepSeek untuk reasoning, bisa swap ke provider lain
def get_llm(temperature: float = 0.1):
    return ChatOpenAI(
        model="deepseek-chat",
        api_key=os.getenv("DEEPSEEK_API_KEY"),
        base_url="https://api.deepseek.com/v1",
        temperature=temperature
    )

class ChatRequest(BaseModel):
    message: str
    temperature: float = 0.1

class ChatResponse(BaseModel):
    answer: str
    model: str
    duration_ms: int

@app.get("/")
async def health(): return {"status": "ok"}

@app.post("/chat", response_model=ChatResponse)
async def chat(req: ChatRequest):
    start = time.time()
    try:
        llm = get_llm(req.temperature)
        res = await llm.ainvoke([HumanMessage(content=req.message)])
        return ChatResponse(
            answer=res.content, model="deepseek-chat",
            duration_ms=int((time.time()-start)*1000)
        )
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

⭐ Model Context Protocol

★ Baru Ditambah Advanced Tool Use

MCP adalah standar terbuka (dibuat Anthropic) yang mendefinisikan bagaimana AI model terhubung ke tools dan data sources secara aman dan konsisten. Analoginya: MCP adalah "USB-C untuk AI" — satu standar, bisa colok ke berbagai tool.

📖 Apa itu MCP?
Definisi Sederhana

MCP adalah protokol yang memungkinkan LLM untuk membaca file, memanggil API, menjalankan code, dan berinteraksi dengan layanan eksternal — melalui antarmuka yang terstandarisasi. Tanpa MCP, setiap tool butuh integrasi custom yang berbeda-beda. Dengan MCP, semua tool bicara bahasa yang sama.

AI Model (Claude / LLM)
│ ← MCP Protocol →
MCP Serverfilesystem / github / browser / API
│
Actual Resourcefiles di disk / repo / web / database
💡 Why It Matters
Tanpa MCP
  • Custom integration per tool
  • Security tidak konsisten
  • Sulit di-maintain
  • Tidak portable antar model
Dengan MCP
  • Satu standar, banyak tool
  • Permission model yang jelas
  • Portable: Claude, GPT, Ollama
  • Community server ecosystem
📁 Filesystem MCP

Memberi AI akses baca/tulis ke direktori tertentu di sistem kamu. Paling dasar dan paling sering dipakai.

bash — Install
npm install -g @modelcontextprotocol/server-filesystem
json — claude_desktop_config.json

  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [

⚠️ Prinsip Least Privilege

Jangan beri akses ke seluruh home directory. Tunjuk ke folder project spesifik saja. AI tidak perlu akses ke ~/.ssh, ~/.aws, atau folder sensitif lainnya.

🐙 GitHub MCP

Buat GitHub Personal Access Token dengan scope minimal: repo:read untuk baca, tambahkan repo:write hanya jika benar-benar butuh push. Jangan pakai token dengan full access.

🔗 APIs via MCP

MCP bisa wrap API eksternal apapun — Notion, Slack, database, atau REST API custom kamu sendiri.

python — custom MCP server sederhana
from mcp.server.fastmcp import FastMCP
import httpx

mcp = FastMCP("My API Server")

@mcp.tool()
async def get_weather(city: str) -> str:
    """Ambil data cuaca untuk kota tertentu."""
    async with httpx.AsyncClient() as client:
        resp = await client.get(f"https://wttr.in/{city}?format=3")
        return resp.text

if __name__ == "__main__":
    mcp.run()
🤝 Relasi MCP dengan Agents
MCP adalah "Tangan" untuk Agent

Agent (seperti OpenCode) adalah otak yang memutuskan apa yang harus dilakukan. MCP adalah tangan yang melakukannya. Tanpa MCP, agent hanya bisa generate teks. Dengan MCP, agent bisa benar-benar berinteraksi dengan dunia: baca file, panggil API, push kode, kirim notifikasi.

KomponenPeranContoh
LLMReasoning & planningQwen, Claude, GPT
Agent FrameworkOrchestrate tool callsOpenCode, LangChain
MCP ServerProvide toolsfilesystem, github, browser
MCP ClientConnect agent ke serverBuilt-in di OpenCode/Claude

⭐ Trust Model — Jangan Blindly Trust

★ Baru Ditambah Critical Mindset

Setelah megang AI secara langsung, kamu akan sadar: AI bisa terlihat sangat yakin padahal salah total. Section ini membahas bagaimana membangun sistem AI yang bisa dipercaya — bukan dengan percaya buta, tapi dengan verifikasi yang terstruktur.

👻 Hallucination
Apa itu Hallucination?

AI menghasilkan output yang salah tapi terdengar meyakinkan. Bukan karena AI "berbohong" — tapi karena model language pada dasarnya adalah prediksi token berikutnya. Ia tidak "tahu" apakah outputnya benar.

Tipe HallucinationContohBahaya Level
Fakta salahLibrary yang tidak ada: import langchain.rag.v2Medium — error saat run
API tidak adaMethod yang tidak exist di docsHigh — silent bug
Logic errorKode jalan tapi hasil salahVery High — sulit deteksi
Security holeSQL query tanpa sanitasiCritical
Kapan AI Paling Sering Halu?
  • Library versi baru yang belum ada di training data
  • Task yang terlalu spesifik ke domain kamu
  • Ketika model kecil dipaksa reasoning kompleks
  • Context window terlalu panjang — model "lupa" context awal
🔍 Verification

Setiap output AI harus diverifikasi sebelum dipercaya. Ini bukan berarti kamu harus baca semua kode baris per baris — tapi ada layer verifikasi yang bisa diotomasi:

1
Syntax Check (otomatis)
python -m py_compile file.py — pastikan kode setidaknya bisa di-parse Python.
2
Import Check (otomatis)
python -c "import module" — pastikan semua import bisa resolve. Kalau error, library mungkin tidak exist atau nama typo.
3
Unit Test (semi-otomatis)
Minta AI juga tulis test untuk kode yang dibuatnya. Jalankan test — output yang lulus test jauh lebih bisa dipercaya.
4
Manual Review (untuk critical code)
Kode yang menyentuh auth, database, file deletion, atau external API — selalu baca manual sebelum jalankan.
🎯 Deterministic Output
Prinsip

Untuk task production, desain sistem sehingga output AI bisa diverifikasi secara deterministik. Jangan andalkan AI untuk output yang tidak bisa di-validate secara otomatis.

python — contoh: paksa output terstruktur
from pydantic import BaseModel, ValidationError

class ExtractedData(BaseModel):
    product_name: str
    price: float
    in_stock: bool

# Minta AI output JSON, lalu validasi dengan Pydantic
try:
    data = ExtractedData.model_validate_json(ai_output)
    # Kalau sampai sini, output valid ✓
except ValidationError as e:
    # Output tidak valid — jangan dipakai, retry atau escalate
    print(f"AI output invalid: {e}")
👤 Human Approval
Human-in-the-Loop

Untuk aksi yang tidak bisa di-undo (hapus data, kirim email, push ke production, charge kartu kredit), selalu tambahkan step persetujuan manusia — bahkan di sistem yang sudah "terpercaya".

AksiPerlu Approval?
Generate teks, kodeTidak — baca dan review saja
Baca file, databaseTidak — read-only aman

👁️ Observability

Kamu tidak bisa mempercayai sistem yang tidak bisa kamu observe. Minimal harus ada:

  • Input logging — apa yang masuk ke model (prompt, context)
  • Output logging — apa yang keluar dari model
  • Latency tracking — berapa lama setiap call
  • Error logging — kapan dan kenapa gagal
  • Token usage — berapa token dipakai (untuk cost tracking)

Daftar di smith.langchain.com, set environment variable LANGCHAIN_TRACING_V2=true — semua call LangChain otomatis tercatat di dashboard. Kamu bisa lihat input, output, latency, dan error dari setiap call.

Eval & Observability

✗ Belum Ada★ Engineer vs Hobbyist
python — eval_rag.py
from dataclasses import dataclass

@dataclass
class TestCase:
    question: str
    expected_keywords: list[str]

tests = [
    TestCase("Apa itu Ollama?", ["ollama", "local", "model"]),
    TestCase("RAM untuk 7B?", ["16gb", "ram"]),
]

def evaluate(chain, tests):
    scores = []
    for tc in tests:
        answer = chain.invoke(tc.question)
        score = sum(1 for kw in tc.expected_keywords
                     if kw.lower() in answer.lower()) / len(tc.expected_keywords)
        scores.append(score)
        print(f"Q: {tc.question} | Score: {score:.0%}")
    print(f"\nAverage: {sum(scores)/len(scores):.0%}")
